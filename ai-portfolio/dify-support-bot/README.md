# 🤖 HashKey Exchange 智能客服助手

> 基于 Dify + 父子分段知识库 + 双层检索架构构建的持牌交易所 AI 客服系统。

## 解决什么问题

HashKey Exchange 是香港持牌虚拟资产交易所，用户每天有大量关于费率、VIP 等级、充提规则、合规限制的咨询。传统人工客服面临响应慢、一致性差、合规压力大等问题。

## 知识库设计思路

### 双层知识库架构
- **第一优先级**：高频 FAQ 库（标准问答对，命中即返回）
- **第二优先级**：10 份精准文档（按问题类型映射）
- **第三优先级**：引导用户查阅官方公告或转人工

### 父子分段策略（核心亮点）

| 方案 | 问题 | 我们的方案 |
|------|------|-----------|
| 传统分段 | 长块带回无关内容；短块丢失表格上下文 | ❌ |
| **父子分段** ✅ | **子块**精准命中，**父块**带回完整表格上下文 | ✅ |

- **父分段**：按二级标题(##)切分，800-1000 字符
- **子分段**：按句号(。)切分，200-300 字符
- **实测效果**：费率问题检索准确率提升约 27%，表格完整性接近 100%

### 表格预处理
所有含表格的文档通过 iLovePDF 转换为 Excel(.xlsx) 格式后再上传，降低表格断裂风险。

**用户：** VIP3费率是多少？
